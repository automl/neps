    NePS-Spaces: Joint Architecture and HyperParameter Search (JAHS)#

    NePS-Spaces provides a powerful framework for defining and optimizing complex search spaces, enabling both and joint architecture and hyperparameter search (JAHS).

    How to use NePS-Spaces#

    A NePS space include all the necessary components to define a Hyperparameter Optimization (HPO) search space, including:

    • Integer: Discrete integer values
    • Float: Continuous float values
    • Categorical: Discrete categorical values
    • Fidelity: Special type for float or integer, multi-fidelity parameters (e.g., epochs, dataset size)

    def centered_around(
    self,
    center: int,
    confidence: ConfidenceLevel,
) -> Integer:
    """Create a new integer domain centered around a specific value.

    Args:
        center: The value around which to center the new domain.
        confidence: The confidence level for the new domain.
        center: int:
        confidence: ConfidenceLevel:

    Returns:
        A new Integer instance that is centered around the specified value.

    Raises:
        ValueError: If the center value is not within the domain's range.

    """
    new_min, new_max = cast(
        tuple[int, int],
        _calculate_new_domain_bounds(
            number_type=int,
            min_value=self.min_value,
            max_value=self.max_value,
            center=center,
            confidence=confidence,
        ),
    )
    return Integer(
        min_value=new_min,
        max_value=new_max,
        log=self._log,
        prior=center,
        prior_confidence=confidence,
    )

    def sample(self) -> float:
    """Sample a random floating-point value from the domain.

    Returns:
      A randomly selected floating-point value within the domain's range.

    Raises:
      ValueError: If min_value is greater than max_value.

    """
    if self._log:
        log_min = math.log(self._min_value)
        log_max = math.log(self._max_value)
        return float(math.exp(random.uniform(log_min, log_max)))
    return float(random.uniform(self._min_value, self._max_value))

    def centered_around(
    self,
    center: int,
    confidence: ConfidenceLevel,
) -> Categorical:
    """Create a new categorical domain centered around a specific choice index.

    Args:
      center: The index of the choice around which to center the new domain.
      confidence: The confidence level for the new domain.
      center: int:
      confidence: ConfidenceLevel:

    Returns:
      A new Categorical instance with a range centered around the specified
      choice index.

    Raises:
      ValueError: If the center index is out of bounds of the choices.

    """
    new_min, new_max = cast(
        tuple[int, int],
        _calculate_new_domain_bounds(
            number_type=int,
            min_value=self.min_value,
            max_value=self.max_value,
            center=center,
            confidence=confidence,
        ),
    )
    new_choices = cast(tuple, self._choices)[new_min : new_max + 1]
    return Categorical(
        choices=new_choices,
        prior_index=new_choices.index(cast(tuple, self._choices)[center]),
        prior_confidence=confidence,
    )

    Defining a JAHS Search Space#

    You define a JAHS space by composing Pipeline, Operation, and Domain objects. This allows you to jointly search over architectures and hyperparameters.

    Example: Simple CNN Architecture + Hyperparameters#

    from neps.space.neps_spaces.parameters import Pipeline, Operation, Float, Integer, Categorical, Fidelity

# Define operations (e.g., convolution, pooling, activation)
conv_op = Operation(
    name="conv",
    kernel_size=Categorical([3, 5]),
    filters=Integer(16, 64),
    activation=Categorical(["relu", "tanh"]),
)
pool_op = Operation(
    name="maxpool",
    pool_size=Categorical([2, 3]),
)

# Define the pipeline (architecture)
pipeline_space = Pipeline(
    layers=[conv_op, pool_op],
    learning_rate=Float(1e-4, 1e-2, prior=1e-3),
    optimizer=Categorical(["adam", "sgd"]),
    epochs=Fidelity(Integer(1, 100)),  # Mark as fidelity for multi-fidelity
)

    You can nest operations, use lists for repeated blocks, and combine with standard hyperparameters.

    Domains and Priors#

    Each parameter is defined by a Domain object: - Float: Continuous range (optionally log-scale, with prior and confidence) - Integer: Discrete range (optionally log-scale, with prior and confidence) - Categorical: Discrete set of choices (with optional prior)

    You can specify a prior value and a confidence level (low, medium, high) to guide the optimizer.

    Sampling and Resolution#

    NePS-Spaces provides a flexible sampling and resolution system: - Random sampling: Uniformly samples from all domains - Prior-based sampling: Uses priors with a tunable probability - Mutation/crossover: For complex search strategies - Fidelity-aware sampling: For multi-fidelity optimization (e.g., Successive Halving, Hyperband)

    Available Algorithms#

    RandomSearch#

    Samples configurations randomly from the pipeline space.

    ComplexRandomSearch#

    Samples randomly, but also mutates and crosses over previous successful configurations.

    PriorBand (Multi-Fidelity + Prior)#

    Implements the PriorBand algorithm, which combines prior-based, incumbent, and random sampling in a multi-fidelity bracketed setting (e.g., Successive Halving, Hyperband, ASHA).

    Example: Using PriorBand#

    from neps.space.neps_spaces.optimizers.bracket_optimizer import priorband

optimizer = priorband(
    space=pipeline_space,
    eta=3,
    sample_prior_first=True,
    base="hyperband",  # or "successive_halving", "asha", "async_hb"
)

    Multi-Fidelity and Fidelity Parameters#

    To enable multi-fidelity optimization, mark a parameter as a Fidelity: 

    from neps.space.neps_spaces.parameters import Fidelity, Integer

pipeline_space = Pipeline(
    ...,
    epochs=Fidelity(Integer(1, 100)),
)
    This allows algorithms like PriorBand to efficiently allocate resources.

    Advanced Sampling: Mutation, Crossover, and Priors#

    NePS-Spaces supports advanced sampling strategies: - MutateByForgettingSampler: Randomly forgets some choices to encourage exploration - CrossoverByMixingSampler: Mixes two configurations - PriorOrFallbackSampler: Uses prior with a given probability, otherwise falls back to another sampler

    These can be used to build custom optimizers or for research.
